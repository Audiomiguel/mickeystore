GITHUB REPOSITORY LINKEDIN Acerca del proyecto Angular

Se ha utilizado la comunicación entre componentes padre a hijo y viceversa (Input & Output) Por ejemplo: El componente Products, cual posee una lista de Productos, se comunica al componente Product para enviar una interfaz de tipo Product.

Se cumplió con el ciclo de vida de un Componente, el constructor para la inyección de dependencias, ngOnChanges para detectar los cambios de los Inputs en los formularios y ngOnInit para el uso de API requests,

Lazy Loading

Reduce el peso de la aplicación principal modularizando los componentes, los cuales serán llamados cada vez que sean necesarios y no cargados desde un principio.

UI,UX, Estilos Angular Material

Se utilizó Angular Material para el uso de Cards, tipografías, componentes ded diseño, formularios, paleta de colores, etc.
